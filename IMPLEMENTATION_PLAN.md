# 胸带式激光随跑"兔子" - 5天原型开发方案

## 项目概述

**核心功能**：胸带设备通过舵机控制激光投影，在地面显示一个以目标速度移动的"虚拟兔子"，引导跑者维持目标配速。

**系统架构**：
```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│  Web/App    │◄────────┤   后端服务   │◄────────┤  胸带设备   │
│  用户界面   │  WiFi   │  (数据处理)  │  BLE    │  (嵌入式)   │
└─────────────┘         └──────────────┘         └─────────────┘
                               │
                               ├─► AI语音服务
                               ├─► 游戏化逻辑
                               └─► 数据持久化
```

## 技术栈推荐

### 嵌入式
- **MCU**: ESP32-S3（蓝牙+WiFi双模）
- **开发框架**: PlatformIO + Arduino Framework
- **编程语言**: C/C++
- **传感器库**: TinyGPS++, Adafruit LSM6DS3, Adafruit BLE

### 后端
- **Web框架**: FastAPI（异步支持）
- **实时通信**: WebSocket
- **BLE库**: bleak（Python）
- **数据库**: PostgreSQL + Redis（缓存）
- **TTS服务**: Azure Cognitive Services / 离线：pyttsx3
- **部署**: Docker + Docker Compose

### 前端
- **框架**: React / Vue 3
- **地图**: Leaflet / Mapbox
- **图表**: ECharts / Chart.js
- **UI组件**: Ant Design / Element Plus
- **状态管理**: Pinia / Zustand

## 数据流设计

### 实时控制数据流
```
胸带设备 (BLE) → 后端WebSocket → 前端实时更新
     ↓
  本地PID控制
     ↓
  舵机角度调整
     ↓
  激光投影位置
```

### 语音提示数据流
```
胸带设备 → 后端 (数据分析) → 触发判断 → TTS服务 → 音频推送到前端
```

### 游戏化数据流
```
运动数据存储 → 积分计算 → 成就解锁 → 数据库更新 → 前端展示
```

## 风险与备选方案

| 风险 | 应对措施 |
|------|----------|
| GPS精度不足 | 多卫星系统（GPS+北斗+GLONASS）+ IMU融合 |
| 蓝牙延迟 | 使用BLE 5.0+，降低数据发送频率，本地PID闭环 |
| 舵机响应慢 | 选择高速舵机，优化控制算法，预测控制 |
| 激光投影可见性 | 使用高亮度激光，选择合适波长（绿色532nm） |
| 电池续航 | 低功耗设计，可更换电池方案 |
| 5天时间不足 | MVP优先，先实现核心功能，游戏化功能后续迭代 |
